# КИМ-2.1. Устные доклады

Устный доклад проверяет и развивает умения студента собирать и систематизировать профессиональную информацию из различных источников. На доклад отводится 5-7 минут на лекционном или практическом занятии, после чего следует обсуждение с преподавателем и группой, где участники могут задавать вопросы, дополнять и корректировать информацию из сообщения. При сборе информации студент может использовать любые методы включая ИИ-ассисстентов, но ответственность за полноту и точность собранной информации ложится на самого докладчика.

Устные доклады модуля 7 фокусируются на лучших практиках работы и применении пройденных инструментов: RestAPI, Swaggeer, FastAPI. 

Критерии оценки:
- полнота покрытия применения инструмента - 1 балл
- общее качество остальных разделов и чистота изложения - 1 балл.

Студент может выбрать тему из предложенных или согласовать свою тему с преподавателем.

Предложенные темы докладов:

1. Принципы проектирования качественного REST API
2. Лучшие практики- проектирования URL и эндпойнтов
3. Выбор HTTP-методов при проектировании API
4. Использование HTTP-статус-кодов: рекомендации и типичные ошибки
5. Версионирование REST API
6. Документирование API как часть процесса разработки программного обеспечения
7. OpenAPI как стандарт описания веб-API
8. Swagger и другие инструменты документирования API
9. Валидация данных в веб-приложениях: назначение и основные подходы
10. Типизация данных в современных Python-фреймворках
11. Принципы обработки ошибок в REST API
12. Единообразие ответов API и стандартизация форматов данных
13. JSON как стандарт обмена данными между клиентом и сервером
14. Асинхронное программирование в веб-разработке: преимущества и ограничения
15. Когда асинхронность действительно необходима в веб-приложениях
16. FastAPI среди современных Python-фреймворков для разработки API
17. Критерии выбора веб-фреймворка для Python-проектов
18. Архитектура современных backend-приложений
19. Принципы организации структуры backend-проекта
20. Разделение ответственности в серверных приложениях
21. Middleware: назначение и области применения
22. Основные требования к безопасности REST API
23. Типичные ошибки при проектировании REST API
24. Производительность веб-API и факторы, влияющие на нее
25. Масштабируемость REST API
26. Тестирование REST API: основные подходы и инструменты
27. REST API и GraphQL: сравнительный обзор
