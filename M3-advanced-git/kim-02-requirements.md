## Требования к репозиториям для 1й части лабораторной работы

Репозитории для этой части, как правило, содержат один файл программного кода и воспроизводят необходимую ситуацию для выполнения действия.

При этом в репозиториях специально создаются пары конфликтующих коммиты, под которыми понимаются коммиты меняющие одну и ту же строчку кода либо соседние строчки кода. Для удобства проверки ответа конфликтующие изменения должны быть независимыми, то есть допускать выполнение в любом порядке.

## Копирование коммитов между ветками

Репозиторий содержит не менее 3х веток, каждая из которых содержит не менее 3х коммитов. Каждая пара веток должна содержать конфликтные коммиты.

##  Удаление изменений

Репозиторий содержит линейную структуру из 5-7 коммитов. Может содержать конфликтующие пары коммитов, чтобы студенты проработали специфический конфликт при удалении близких изменений.

Удаление может происходить как полное, так и с созданием параллельного коммита.

## Линеаризация репозитория

Репозиторий содержит не менее 3х веток, каждая из которых содержит не менее 3х коммитов. Каждая пара веток должна содержать конфликтные коммиты. 

## Идеализация истории

Задачи этого типа выполняются на репозитории линейной структуры. За один раз может выполняться одна или несколько задач. При выполнении одной задачи, рекомендуется не менее 5 коммитов и одна пара конфликтных, при более 1й задачи, не менее 10 коммитов и две пары конфликтных.

### Перемещение коммитов

Перемещение обычно делается так чтобы пара конфликтующих коммитов поменялась местами, чтобы студенты проработали разрешение специфического конфликта при перестановке изменений.

На практике, эта задача часто совмещается со следующей (переместить коммит к тому, который он исправляет, и слить с ним).

### Слияние коммитов

Не требует дополнительных условий.

### Разбиение коммитов.

Коммит для разбиения должен менять несколько строчек. Эти изменения могут быть в разных ханках (легче) или в одном ханке (сложнее).
