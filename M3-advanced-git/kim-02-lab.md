# КИМ-2.1. [Лабораторная работа]

Состоит из двух частей:

1 часть. Внесение изменений в репозиторий для улучшения его качества. Внесение изменений может включать по отдельности или вместе следующие изменения: 
1. копирование коммитов между ветками, 
2. удаление изменений, 
3. линеаризацию репозитория, 
4. идеализацию истории: 
    1. перемещение коммитов (в том числе через конфликтующие коммиты),
	2. слияние коммитов
	3. разбиение коммитов.
	
При этом запланированы типовые конфликты при выполнении действий и студенты должны их разрешить.

Базовые репозитории для этой части разрабатывают студенты на основе требований, при этом при выполнении студенты обмениваются репозиториями и выполняют работу по репозиторию другого человека.

Критерии оценивания:
* корректность выбора комманд Git и определения их параметров - 0-2 балла
* корректность разрешения конфликтов - 0-2 балла

2 часть. Ролевая игра по совместной разработке программного кода. 

Группа студентов играет роль команды разработчиков, которой необходимо внести изменения в большой репозиторий работая одновременно. Изменения могут вносится как в главную ветку разработки (dev, main, master) симулируя срочные багфиксы работающего кода, так и в ветку разработчика, симулируя новую работу). Возможны разные workflow (например, единый центральнй репозиторий или отдельные репозитории разработчиков). При этом каждый студент находится как в роли исполнителя, так и в роли руководителя для другого студента, работу в чьей ветке он ревьювит и сливает в главную. Задания на изменения в коде формулируются элементарные (например "измени пределы цикла на строке такой-то"), но сжатое время выполнения задания (40 минут) используется для создания стресса и симуляции ситуации обычной разработки когда все внимание в коде и на корректное выполнение операций в системе контроля версий его остается мало. После выполнения, преподаватель разбирает с группой получившийся репозиторий и сделанные в нем ошибки.

Критерии оценивания:
корректность в роли руководителя - 0-2 балла 
корректность в роли исполнителя - 0-1 балл
