# КИМ-8.2. Лабораторное занятие

## Цель работы

Освоить принципы контейнеризации и развертывания веб-приложений с использованием Docker, Docker Compose и виртуальных машин, изучить подходы к управлению многоконтейнерными приложениями, а также получить практические навыки построения базового CI/CD-пайплайна с использованием GitHub Actions.

## Формируемые результаты обучения

В результате выполнения лабораторной работы обучающийся должен уметь:

* анализировать структуру готового веб-приложения и его зависимости;
* создавать Docker-образы с использованием Dockerfile;
* описывать многоконтейнерные приложения с использованием Docker Compose;
* настраивать взаимодействие контейнеров приложения, базы данных и вспомогательных сервисов;
* выполнять локальный запуск и проверку контейнеризированного приложения;
* развертывать контейнеризированные приложения на удаленной виртуальной машине;
* проверять доступность и корректность работы развернутого сервиса;
* подготавливать инструкции по развертыванию программной системы;
* создавать базовые CI/CD workflow с использованием GitHub Actions;
* настраивать автоматический запуск проверок при изменении основной ветви разработки.

## Исходные данные

Обучающемуся предоставляется исходный код готовой веб-системы, включающей backend-приложение, базу данных и базовый frontend, репозиторий GitHub, конфигурационные файлы проекта, а также доступ к виртуальной машине для выполнения развертывания.

## Задание

### Часть 1. Контейнеризация и развертывание веб-приложения

1. Выполнить анализ структуры предоставленного программного решения:

   * определить состав компонентов системы;
   * изучить зависимости backend, frontend и базы данных;
   * определить способы взаимодействия между сервисами приложения.

2. Подготовить конфигурацию контейнеризации приложения:

   * разработать или адаптировать Dockerfile для необходимых компонентов системы;
   * определить используемые базовые образы;
   * настроить установку зависимостей и запуск приложений внутри контейнеров.

3. Настроить многоконтейнерное окружение с использованием Docker Compose:

   * описать сервисы приложения в файле `docker-compose.yml`;
   * настроить взаимодействие backend, frontend и базы данных;
   * определить необходимые сети, тома (volumes) и переменные окружения;
   * настроить публикацию используемых портов.

4. Выполнить сборку Docker-образов и запуск системы локально:

   * выполнить сборку образов приложения;
   * запустить контейнеры средствами Docker Compose;
   * проверить состояние запущенных контейнеров;
   * убедиться в корректности взаимодействия компонентов системы.

5. Выполнить тестирование работоспособности контейнеризированного приложения:

   * проверить доступность веб-интерфейса приложения;
   * выполнить проверку API-эндпойнтов (при наличии), включая контрольный запрос к служебному endpoint проверки состояния системы;
   * проверить корректность работы взаимодействия приложения с базой данных.

6. Выполнить развертывание системы на удаленной виртуальной машине:

   * подготовить окружение виртуальной машины;
   * выполнить передачу исходного кода или конфигурации развертывания;
   * запустить приложение средствами Docker Compose в удаленной среде;
   * проверить доступность развернутого приложения по сетевому адресу виртуальной машины.

7. Подготовить инструкцию по развертыванию системы, включающую:

   * требования к окружению;
   * порядок установки необходимых инструментов;
   * описание настройки переменных окружения;
   * команды сборки и запуска приложения;
   * порядок проверки работоспособности системы.

---

### Часть 2. Настройка CI/CD-пайплайна с использованием GitHub Actions

1. Проанализировать процесс разработки проекта и определить проверки, которые должны выполняться автоматически перед слиянием изменений.

2. Создать workflow GitHub Actions:

   * разместить конфигурацию CI/CD в каталоге `.github/workflows`;
   * настроить запуск workflow при создании Pull Request и/или слиянии изменений в ветку `main`;
   * настроить автоматическое получение исходного кода проекта и установку зависимостей.

3. Реализовать базовые этапы автоматической проверки проекта:

   * выполнение проверки качества кода (lint);
   * выполнение тестирования приложения;
   * выполнение дополнительных проверок проекта при необходимости.

4. Выполнить проверку работы CI/CD-пайплайна:

   * создать Pull Request с изменениями;
   * убедиться в автоматическом запуске GitHub Actions;
   * проверить успешное выполнение всех этапов workflow.

5. Настроить правила защиты основной ветви репозитория:

   * включить обязательное прохождение CI-проверок перед слиянием;
   * обеспечить выполнение процесса проверки изменений через Pull Request.

6. Оформить результаты выполнения лабораторной работы:

   * представить описание созданной контейнерной инфраструктуры;
   * приложить инструкцию по развертыванию;
   * описать реализованный CI/CD-процесс;
   * подтвердить корректность запуска и проверки приложения.

## Критерии оценивания

**Максимальная оценка за выполнение лабораторной работы составляет 7 баллов.**

Итоговая оценка определяется на основании суммарного процента выполнения всех этапов лабораторной работы с последующим пропорциональным переводом в балльную систему (7 баллов соответствует 100% выполнения).

| **2. Локальное тестирование и развертывание системы** | Выполнен запуск контейнеризированного приложения, проверена работоспособность компонентов системы, выполнено развертывание на виртуальной машине и подтверждена доступность приложения. |                  **30%** |

> **Примечание.** Каждый этап лабораторной работы оценивается независимо. Итоговая оценка определяется суммой набранных процентов по всем этапам с последующим переводом в баллы.
